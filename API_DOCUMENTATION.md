# QCMS API Documentation

All API requests must include the `Authorization: Bearer <JWT_TOKEN>` header unless otherwise specified.

## 🔐 Authentication & Security

| Method | Endpoint | Description | Public |
| :--- | :--- | :--- | :--- |
| `POST` | `/api/auth/login` | Authenticate user & receive JWT token. | Yes |
| `POST` | `/api/auth/register-org` | Step 1: Register organization and admin user. | Yes |
| `GET` | `/api/auth/me` | Retrieve current user profile. | No |
| `PUT` | `/api/auth/change-password` | Update current user's password. | No |

| Method | Endpoint | Description | Roles |

## 📂 Project Management

| Method | Endpoint | Description |
| :--- | :--- | :--- |
| `GET` | `/api/projects` | List projects for the current user's role. |
| `POST` | `/api/projects` | Create a new project (Auto-assigns Stage 1). |
| `GET` | `/api/projects/<id>` | Detailed project payload with all completed stage data. |
| `DELETE` | `/api/projects/<id>` | Soft-delete a project (Admin & Creator only). |
